# 游戏玩法规则（Draft）

本文档描述 `WS Arena` 的玩法与胜负规则。文内标签含义如下：

- **【已实现】**：本仓库 **sync-evolution-demo** 前后端 demo 中已落地，可与代码对照。
- **【部分实现】**：已有相关数据/UI/掉落等，但机制未按草案写全。
- **【未实现】**：仍为设计目标，demo 中无或未接入权威逻辑。

---

## 基本设定

- **对局限时**：4 分钟 **【未实现】**（服务端未接胜负计时；参数保留在文档供后续接入）
- **开局装备**：默认仅持有 **匕首**，占 1 格 **【已实现】**

---

## 操作与输入 **【已实现】**

- **移动**：`WASD`
- **瞄准**：鼠标相对于对战画布移动；瞄准坐标发往服务端 **【已实现】**
- **攻击**：`空格` 或对战画布指针按下/抬起；按下与抬起时会 **额外立即 flush 一次输入**，避免短按未被 50ms 节拍采样 **【已实现】**
- **拾取**：`E`，拾取距离由服务端校验 **【已实现】**
- **丢弃**：`G`，丢弃 **当前选中格**；匕首不可丢弃 **【已实现】**
- **物品栏**：共 **8 格**；点击或 **`1`–`8`** 选中 **【已实现】**
- **武器使用口径**：**当前选中格** 决定攻击使用匕首挥击还是枪械射击 **【已实现】**

### 规划中的操作（对照 API，便于后续接入）

- **武器槽位**：主武器 / 副武器 / 匕首三分法 **【未实现】**（当前为统一 8 格物品栏）
- **循环切换**：`Q` 在三槽间循环 **【未实现】**
- **快速直切**：`1` 主武器、`2` 副武器、`3` 匕首 **【未实现】**（当前 `1`–`8` 对应 8 格）
- **鼠标滚轮切换武器** **【未实现】**

## 匕首（近战） **【已实现】**

- **攻击范围**：以朝向为中心，左右各 60°（合计 **120°**）扇形；距离与伤害由服务端判定 **【已实现】**
- **丢弃**：匕首为默认武器，**不可丢弃** **【已实现】**
- **客户端表现**：挥击时绘制 **120° 扇形半月刀光**；命中时另有命中反馈 **【已实现】**
- **攻击间隔**：冷却帧数由服务端常量约束 **【已实现】**（具体数值见代码）

## 枪械与弹药 **【已实现】**

| 规则 | 说明 |
|------|------|
| 武器格弹药 | 枪械占 1 格，该格 **`q` = 枪内剩余弹药** **【已实现】** |
| 弹匣容量（demo） | SMG：`30`；AR：`30`（与服务端 `maxStack` 一致）**【已实现】** |
| 开火消耗 | 只扣 **当前选中枪械格** 的 `q` **【已实现】** |
| 地图掉落枪 | 落地枪械带 **满匣**（如 SMG `q = 30`）**【已实现】** |
| 弹药包堆叠 | `ammo_9mm` / `ammo_762` 可堆叠；拾起时 **先装入未满的同口径枪械弹匣**，余量再进堆叠格 **【已实现】** |

以下为 **历史草案表述**（保留语义，避免与「枪格自带弹量」混淆）：

- **耗尽即停火**：枪格 `q = 0` 且无堆叠弹药可装填时无法射击 **【已实现】**
- **旧草案「射击从背包弹药堆直接扣一发」**：与本 demo 不符；以「**只扣武器格 q**」为准 **【已实现】**

## 自动换弹 **【已实现】**

- **触发**：当前格枪械 **打空**，且背包内仍有 **对应弹种堆叠**（`ammo_9mm` / `ammo_762`）时 **自动进入换弹**；空枪再次尝试攻击亦可触发 **【已实现】**
- **时长**：`RELOAD_DURATION_FRAMES`（当前 **45 帧** × tick 间隔）**【已实现】**
- **完成**：从堆叠装入该枪械格至满或堆叠用尽；死亡/开局清除换弹状态 **【已实现】**

## 网络同步现状（2026-04） **【已实现 / 部分实现】**

### 当前结论

- 当前实现已经比早期 demo 更接近真实实时对战服务器，但仍属于「高质量原型/小规模可用」，尚未到完整商用网络架构。

### 已接近真实项目的点

- **输入上报**：已从无脑高频改为「变化触发 + 节流上限 + 持续按键保活心跳」。
- **状态下发**：已从每帧全量演进为「快照 + 增量」混合：
  - `snapshot` 保持全量（兜底）
  - `state` 对 `players / bullets / drops` 走 delta（含 removed ids）
- **服务端权威**：命中判定与战斗结果仍以服务端为准。

### 与典型商用方案的差距

- **传输层**：当前为 WebSocket 单通道；大型项目常做通道分级与更细粒度可靠性策略。
- **兴趣管理**：尚未做视野/距离裁剪（interest management）。
- **快照体系**：已具备基础 delta，但未做完整 ack/重传/历史缓冲策略。
- **大规模优化**：序列化压缩、对象池与高并发场景优化仍可继续加强。
